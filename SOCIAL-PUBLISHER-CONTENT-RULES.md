# Social Publisher Content Rules

This file defines deterministic pre-publish checks for Josh's social publisher. It is intentionally human-authored and version-controlled. It does not require an AI assistant.

## Writing checks

1. Enable the browser's spellcheck for the caption and alternative-text fields.
2. Review spelling, grammar, capitalization, and punctuation before selecting final review.
3. Use plain language and short paragraphs. Prefer one idea per paragraph.
4. Do not use em dashes in customer-facing copy.
5. Keep calls to action specific: shop, follow, view, or learn more.
6. Confirm dates, prices, launch timing, product quantities, and links manually before publishing.

## Furious Facades voice

- Handmade, strange, cinematic, and confident.
- Describe materials and process accurately. Do not make claims that cannot be verified.
- Use horror atmosphere without graphic shock language.
- Refer to the antlered creature as a Wendigo.

## Required review checklist

Before a post can be marked ready for publishing, the operator confirms:

- Destination accounts are correct.
- Caption and image alternative text have been proofread.
- Each selected platform preview has been checked for crop, legibility, and link placement.
- The media is owned by Furious Facades or cleared for use.
- The scheduled time is correct in Eastern Time when scheduling is selected.

## Image delivery rules

- Facebook: landscape delivery crop.
- Instagram: portrait feed delivery crop.
- TikTok: vertical delivery crop.
- Threads: square delivery crop.
- X: wide delivery crop.
- LinkedIn: landscape delivery crop.

The production app should preserve the uploaded original, generate a separate optimized derivative for each selected destination, and log the generated file names with the draft. Never alter the original file in place.

## Change control

Update this document when Josh changes brand voice, review requirements, or delivery rules. Platform API specifications and hard limits must be verified from official documentation during production implementation because they change over time.
